# Canvas HiDPI 캔들 차트 비교 데모 — 스펙

## 1. 목적

HTML Canvas로 캔들 차트를 그릴 때, **devicePixelRatio(DPR)를 적용하지 않은 경우**와 **적용한 경우**의 차이를 한 페이지에서 나란히 비교한다.

특히 HiDPI(Retina 등) 환경에서 **최고가·최저가 텍스트와 축 라벨이 흐릿해지는 문제**를 눈으로 확인할 수 있는 간단한 데모다.

## 4. 페이지 구성

### 4.1 상단 영역

- 제목 (한글)
- 한 줄 안내: 확대해 최고가·최저가 글자 선명도를 비교하라는 설명
- 현재 `devicePixelRatio` 값 표시  
  예: `현재 devicePixelRatio: 2`

### 4.2 공통 컨트롤

| 컨트롤 | 동작 |
|--------|------|
| 확대 / 축소 | 활성 차트 기준 확대 / 축소 (동기화 ON이면 양쪽) |
| 리셋 | 초기 뷰(최근 500개)로 복귀 |
| 500개 | 화면에 약 500개 캔들이 보이도록 줌 |
| 40개 | 화면에 약 40개 캔들이 보이도록 줌 |
| 동기화 (체크박스 토글) | ON이면 한쪽 조작이 다른 쪽에도 반영 |
| 지금 맞추기 | 활성 차트 구간을 다른 차트에 한 번 복사 |

**활성 차트 (active)**

- 초기 활성 차트: **DPR 적용** (오른쪽)
- 포인터·휠·패널 로컬 버튼이 발생한 패널이 활성으로 바뀐다
- 각 패널에 `조작 중` 배지로 활성을 표시한다
- 공통 버튼: 동기화 OFF → 활성만, 동기화 ON → 양쪽
- 각 패널에도 로컬 `확대` / `축소` / `리셋`을 둔다
- `지금 맞추기` 기준: 활성 차트 (없으면 DPR 미적용/왼쪽을 폴백)

### 4.3 차트 영역 (반응형)

- **넓은 화면**: 좌우 2열
  - 왼쪽: `DPR 미적용` (일반 Canvas)
  - 오른쪽: `DPR 적용` (devicePixelRatio 보정)
- **좁은 화면**: 위아래 스택 (미적용 → 적용 순)

각 패널:

- 제목 (한글)
- `조작 중` 배지
- 로컬 `확대` / `축소` / `리셋`
- `<canvas>`

## 5. 데이터

### 5.1 생성 방식

- 페이지 로드 시 JavaScript로 **가짜 OHLC 2,000개** 생성
- 랜덤 워크 기반 (자연스러운 상승/하락 구간 포함)
- **시드 고정** (기본 `42`) — 새로고침해도 동일 패턴
- 분봉 가정: `Date.UTC(2024, 0, 1, 9, 0, 0)`부터 1분 간격
- OHLC 제약: `high >= max(open, close)`, `low <= min(open, close)`, `high >= low`

### 5.2 캔들 필드

```ts
{
  time: number,   // Unix ms
  open: number,
  high: number,
  low: number,
  close: number
}
```

### 5.3 초기 뷰

- 전체 2,000개 중 **최근(또는 끝쪽) 500개**가 화면에 보이도록 시작
- 프리셋:
  - **500개**: 약 500개 표시
  - **40개**: 약 40개 표시
- 더 축소하면 전체(최대 2,000개)까지 볼 수 있음

## 6. 차트 렌더링

### 6.1 공통 원칙

- **하나의 공통 렌더 함수** + `useDevicePixelRatio: boolean` 옵션
- 두 차트는 같은 데이터·같은 그리기 로직을 사용
- 차이는 캔버스 버퍼 크기와 `ctx.scale` 적용 여부만

### 6.2 DPR 미적용 (비교용)

- `canvas.width` / `canvas.height` = CSS 픽셀 크기
- `ctx.scale` 없음
- HiDPI에서 텍스트·선이 흐릿할 수 있음

### 6.3 DPR 적용 (보정)

```
dpr = window.devicePixelRatio || 1
canvas.width  = cssWidth  * dpr
canvas.height = cssHeight * dpr
canvas.style.width  = cssWidth  + 'px'
canvas.style.height = cssHeight + 'px'
ctx.setTransform(dpr, 0, 0, dpr, 0, 0)
```

- 이후 그리기는 CSS 픽셀 좌표계로 동일하게 작성

### 6.4 시각 스타일

- 배경: 밝은 톤
- 상승 캔들: 초록
- 하락 캔들: 빨강
- 연한 그리드
- 장식 최소화 (데모 초점 = 텍스트 선명도)

### 6.5 최고가 · 최저가

- 기준: **현재 화면에 보이는 구간의** high 최대 / low 최소
- 표시:
  - 해당 가격의 가로 가이드 라인
  - 가격 텍스트 (예: `최고 12,345.67` / `최저 11,900.12`)
- 줌·팬 시마다 재계산

### 6.6 축

| 축 | 위치 | 동작 |
|----|------|------|
| 가격축 | 오른쪽 | 줌에 따라 눈금 간격·라벨 밀도 조절 |
| 시간축 | 아래 | 가짜 분봉 시각 기준, 줌에 따라 `MM/DD` ↔ `HH:mm` 등 단위·밀도 조절 |

## 7. 인터랙션

두 차트 **기본은 독립** 뷰포트(시작 인덱스, 보이는 개수 등)를 가진다.

### 7.1 줌

| 입력 | 동작 |
|------|------|
| 마우스 휠 | 확대 / 축소 |
| 상하 드래그 (마우스 또는 한 손가락) | 위 = 확대, 아래 = 축소 |
| 두 손가락 핀치 | 확대 / 축소 |
| 확대 / 축소 버튼 | 확대 / 축소 |
| 500개 / 40개 버튼 | 해당 개수로 맞춤 |

줌 중심: 가능하면 **커서/핀치 중심** 기준. 버튼 줌은 현재 뷰 중앙 기준.

### 7.2 팬

| 입력 | 동작 |
|------|------|
| 좌우 드래그 (마우스 또는 한 손가락) | 차트 이동 |

### 7.3 동기화

- **동기화 OFF (기본)**: 각 차트 독립
- **동기화 ON**: 한쪽의 줌/팬/프리셋이 다른 쪽 뷰포트에 동일 반영
- **지금 맞추기**: 활성 차트의 현재 구간을 다른 쪽에 복사 (활성 없으면 왼쪽/`raw` 폴백). 토글 상태와 무관하게 1회 동작
- viewport 변경은 단일 진입점(`applyViewport`)으로 처리해 피드백 루프를 막는다

### 7.4 리사이즈

- 창 크기 변경 시 캔버스 CSS 크기 재측정 후 재렌더
- DPR 적용 차트는 버퍼도 다시 맞춤
- 가능하면 `devicePixelRatio` 변경(모니터 이동 등)에도 버퍼·라벨을 갱신한다

## 9. 기술 제약

- 외부 라이브러리 사용 금지
- ES 모듈 없이도 동작 가능한 단순 구성 권장 (`<script src="chart.js">`)
- `touch-action` 등으로 브라우저 기본 핀치 줌이 차트를 방해하지 않게 처리
- `resize` / `devicePixelRatio` 변경(가능하면)에 대응

## 10. 수락 기준 (Acceptance)

1. `index.html`을 브라우저로 열면 차트 두 개가 한 페이지에 보인다.
2. 반응형: 넓은 화면 좌우, 좁은 화면 세로 스택.
3. 캔들 약 2,000개가 생성되고, 초기에는 약 500개가 보인다.
4. 휠, 상하 드래그, 좌우 드래그, 핀치, 버튼으로 줌/팬이 된다.
5. 보이는 구간의 최고가·최저가가 가이드 라인과 텍스트로 표시된다.
6. 가격축·시간축이 있고, 줌에 따라 눈금이 달라진다.
7. DPR 미적용 / 적용 차트가 같은 그리기 로직을 쓰며, 옵션만 다르다.
8. HiDPI에서 확대 시 DPR 적용 쪽 텍스트가 더 선명하다. (DPR=1 환경에서는 차이가 거의 없을 수 있음 — 상단에 DPR 값 표시로 맥락 제공)
9. 동기화 토글과 「지금 맞추기」가 동작한다.
10. UI 문구가 한글이다.
